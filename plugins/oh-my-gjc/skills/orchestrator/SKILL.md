# Orchestrator (오케스트레이터 모드 — 임계값 기반 위임 정책)

목적: top-level 에이전트의 컨텍스트 예산을 라우팅·위임 프롬프트 설계·통합·검증에 쓰고,
대량 구현은 `executor` 서브에이전트에 위임한다. "무조건 위임"이 아니라 **임계값 기반**이다 —
소형 작업을 위임하면 위임 프롬프트 작성 + 서브에이전트 컨텍스트 재탐색 오버헤드가 직접 수행보다
비싸진다.

## 정책 (적용 시 이 절을 그대로 따른다)

### task tool로 위임
다음 중 하나라도 해당하면 `executor`에 위임한다:
- 변경이 2개 이상 파일에 걸침
- 예상 diff가 ~50줄 초과
- 독립적·병렬 가능한 슬라이스로 나뉨

분석 작업은 인라인으로 수행하지 말고 `planner`(순서 설계), `architect`(아키텍처·코드 리뷰),
`critic`(계획 비평)을 쓴다.

### 직접 수행 (위임 금지 — 오버헤드가 이득을 초과)
- 단일 파일·위치 확정·소규모 수정 (오타, 설정값, 함수 하나)
- 읽기·검색·질문에 대한 직접 답변
- 위임 결과의 통합과 **모든 최종 검증**

### 위임 프롬프트 계약 (모든 task 호출에 필수)
- 정확한 파일 경로 + 충분한 최소 컨텍스트
- 완료 조건과 기대 출력 형태
- 검증 명령: executor에 할당하거나 부모가 보유함을 명시
- 명시적 범위 밖 목록 ("~는 건드리지 말 것")
- 행동 스펙 원천 하나 (기존 테스트 또는 열거된 수용 기준) — 산문 중복 금지

### executor 프롬프트 요령 (Codex 계열, 2026-07-18 실측 근거)
계약형 프롬프트로 codex 프리셋 executor(`gpt-5.6-terra`/`luna` 계열)를 구동하면 단일 수술적
edit로 수렴한다(측정: 동일 과제에서 최소 출력 토큰·edit 1회). 유지 조건:
- 명령문을 선두에 ("Implement X in path/y so that `cmd` passes"), 서사형 서두 금지
- 부정 제약을 문자 그대로 ("do not modify …", "do not create new files")
- 테스트를 스펙으로 지목하고 검증 명령을 executor가 직접 실행·보고하게 지시
- thinking은 medium 기본, 알고리즘적으로 까다로운 슬라이스만 `:high`

### 금지
- "시간 아끼려고" 멀티파일 기능을 인라인 구현 — 이 정책이 막으려는 위반 그 자체
- 위임 결과를 집중 검증 없이 완료 처리
- 자잘한 수정·답변을 위임

## 동작 방식 (GJC 0.11 검증된 메커니즘)

- **세션 적용(`on`)**: 이 스킬 문서가 로드된 현재 세션에만 정책을 적용한다. 파일을 쓰지 않는다.
- **영속 설치(`install`/`user`)**: `bin/install-orchestrator.sh`가 설치한다. GJC 0.11에서
  기본 시스템 프롬프트는 rules의 `alwaysApply`를 렌더링하지 않으므로 상시 정책 텍스트는
  프로젝트 `AGENTS.md`(context file)로 들어가야 하고, `.agents/rules/`는 TTSR 트립와이어
  (첫 edit/write 시점 1회 미드스트림 리마인더)와 `rule://orchestration` 온디맨드 참조를
  담당한다. AGENTS.md 상향 탐색은 git 저장소 루트에서 멈추므로 저장소마다 설치해야 한다.
- **서브에이전트 면제(설계 확인)**: task tool 서브에이전트는 rules와 AGENTS.md를 상속하지
  않는다. 따라서 이 정책은 top-level 오케스트레이터만 구속하고 executor는 할당 범위를
  자유롭게 구현한다.

## 안전 계약

- 파일 쓰기는 `install`(프로젝트)·`user`(사용자 전역) 서브커맨드에서만 일어난다.
  `on`/`status`는 읽기 전용이다.
- `install`은 대상 프로젝트의 `AGENTS.md`에 마커(`## Orchestration policy`) 기반으로
  멱등 append하며, 기존 `AGENTS.md`가 있으면 append 전에 타임스탬프 백업을 만든다.
  마커가 이미 있으면 건너뛴다. 기존 내용을 수정·삭제하지 않는다.
- `user`는 `~/.gjc/agent/rules/`에 규칙 2개만 쓴다. `models.yml`·모델 설정·자격증명·
  다른 사용자 파일은 건드리지 않는다.
- 이 스킬은 위임 판단 기준을 바꿀 뿐, 정확성·검증 의무·승인 경계를 낮추지 않는다.
  특히 "모든 최종 검증은 오케스트레이터가 직접"이 정책의 비타협 조항이다.
